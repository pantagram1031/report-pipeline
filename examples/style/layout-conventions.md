# Example layout conventions

These are optional examples, subordinate to the request and form.

- Keep body-page density reasonably uniform.
- Prefer inline equations when they remain readable; use display equations for
  deliberate emphasis or multi-line derivations.
- Keep table rows compact and allocate more width to explanatory columns.
- Captions should be visually subordinate to body text.
- A heading at the bottom is not automatically a defect; a heading followed by
  a large unexplained void is.
- Validate layout through a contact sheet before inspecting individual pages.
